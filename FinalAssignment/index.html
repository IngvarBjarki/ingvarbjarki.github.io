SOCIAL DATA ANALYSIS

Hi! Our names are Andri and Ingvar, and we have carried out an analysis on the SF311 public service requests dataset from the city of San Francisco, focusing on graffiti complaints.

About the project

It is a well-known fact that people have different opinions regarding graffiti. Is it an art form or vandalism? It is often painted as a scapegoat for all manner of urban ills, ranging from burglary and violence to gentrification. It can at the same time, done right, add color and brighten up dull and ugly buildings or public spaces. We can however most likely all agree that not all graffiti is created equal. We can’t equate well thought out and structured street art to someone’s illegible tag that can hardly have taken more than a minute to spray. This is the kind of graffiti that devalues properties for both individuals and businesses. Property owners can incur large expenses from the negative images that the graffiti generates and the overall community is affected. You can however help reduce graffiti in your community by recording and reporting it.

The city and county of San Francisco established SF311 where residents and businesses can report issues such as vandalism via phone, email, social media or designated mobile applications. All the complaints are gathered in the SFF 311 public service request dataset, updated regularly. This detailed log provides an opportunity to go through all of it, extracting information that would be useful to the municipality to be able to improve its service level. We will use methods ranging from basic statistics to machine learning methods to provide the authorities with useful information to be able to improve the service level. This includes knowing where the vandalism is most likely to take place, how many complaints they can expect at any given time, or what kind of properties are likely to be vandalized.

The Map

Here we have used the machine learning method KNN and K-means which you can toggle between with the drop-down menu. Also, hover over the different districts to see basic information about each and every one of them. After the KNN model is run, predictions are added to these little windows. The KNN predicts whether new complaints are classified as public or private. It also predicts in what district the complaint will relate to. Click on the buttons to decide how many neighbors you want the algorithm to use for the predictions. The algorithm makes a lot of those predictions and then each district is colored based on how many theoretical public vs private property complaints are filed. More blue represents a higher ratio of private property complaints and more red means more public complaints. K-means divides the complaints into K-many clusters, and creates equally many centroids, one for each cluster. The algorithm wants to place the centroids far away from one another, but at the same time minimize the distance to the data points that belong to their clusters. Click the buttons on the graph to choose how many K’s you want to use. The small dots that appear on the graph represent each represent a complaint in from the dataset and are plotted on this graph based on their latitudinal and longitudinal coordinates. They are colored based on the cluster that they are assigned to. The centroid they belong to is the big dot that is the same color.

Multiple Linear Regression

A multiple linear regression model was used to predict the number of graffiti complaints in each week. The input into the model, what it uses to make the predictions are the number of complaints per week that belong to other categories than graffiti, and the total number of complaints by districts per week. It was compared to a naïve model that just always guesses the weekly average and uses that as a prediction. The simplest version of the linear regression model, which was in fact still a simple linear regression model, performs approximately four times as well as the naïve model based on squared error. Take that naïve model! The simple linear regression model used the number of blocked street or sidewalk complaints to predict and was 48.1% accurate. Adding street and sidewalk cleaning gives 65.7% accuracy, and adding noise reports makes it 71.6% accurate. Adding total number of complaints from the Mission district improves accuracy to 77.4%, the district Northern (80.3%) and finally adding the district Park reduced the accuracy slightly (79.4%). Let's try making a prediction using the model that was developed. Say we have 80 blocked sidewalk complaints, 3000 street and sidewalk cleaning complaints, 50 noise complaints, 1500 complaints from the Mission district, 700 from the Northern district, and 500 from Park this week. According to the linear regression model we can then expect the number of graffiti complaints for this week to be 1267.

Decision Tree

A decision tree was used to classify complaints as either public or private property complaints. The input into the model was the hour the complaint is filed, from which district it is filed, as well as the latitudinal and longitudinal coordinates associated with the complaint. To test if how well it performs we created a naïve model that simply always guesses the most common complaint type, which turns out to be 59.7% accurate. The simplest decision tree we created outperforms the naïve model, which is good news. Using only the police district from which the complaints were filed results in a 62.1% accurate model. Adding the hour when the complaints are file results in 62.5% accuracy, and adding latitudinal coordinates makes it 76.5% accurate, and longitudinal coordinates 79.3% accurate. Now for some action! Let’s start predicting with the model. We tell the model that the complaint comes from the Mission district at 3am and has the coordinates (37.764644, -122.426927). Model says: Graffiti private property!

Regression tree

A regression tree was used to predict the time it takes to solve a given case based on the same input that was used in the decision tree model: the hour the complaint is filed, the district it is filed, latitude and longitude. Again, always guessing the mean value is used as a naive benchmark model to evaluate the performance. The simplest form of the regression tree performs about ten times better than the naive model based on squared error. Now let’s use the model to predict the time it takes to solve some specific case. Let’s put the district Mission into the model, complaint time 5pm, let’s say it relates to public property and has the coordinates (37.764641, -122.426929). This would result in a resolve time of 36,098 seconds, or about 10 hours.
